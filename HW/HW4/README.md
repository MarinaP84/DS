Сравнение классификаторов для обнаружения мошеннических транзакций (Credit Card Fraud Detection)
Описание проекта
В данном проекте решается задача бинарной классификации для обнаружения мошеннических транзакций по кредитным картам. Датасет содержит 284 807 транзакций, из которых только 492 (0.17%) являются мошенническими – сильный дисбаланс классов.
Цель работы:
· Реализовать минимум 5 классификаторов из предложенного списка
· Провести гиперпараметрическую настройку для каждой модели
· Сравнить метрики и выбрать лучший классификатор для данного датасета
Используемые классификаторы
№ Модель Тип Балансировка
1 Gradient Boosting Градиентный бустинг sample_weight
2 CatBoost Бустинг (Яндекс) auto_class_weights='Balanced'
3 XGBoost Экстремальный градиентный бустинг scale_pos_weight
4 KNN Метод k ближайших соседей Стандартизация + weights
5 Decision Tree Дерево решений class_weight='balanced'

Лучшая модель
XGBoost показал наилучший результат с F1-score = 0.83 для класса мошеннических транзакций.
Преимущества XGBoost:
· Сбалансированные precision (0.89) и recall (0.78)
· Эффективная работа с несбалансированными данными
· Высокая скорость обучения (8-12 минут)
· Хорошая обобщающая способность (CV F1=0.8558)
Лучшие параметры XGBoost:
```python
{
    'learning_rate': 0.2,
    'max_depth': 5,
    'n_estimators': 200,
    'subsample': 1.0
}
```
Детальный анализ моделей
1. XGBoost
· Recall = 0.78 – находит 78% мошеннических транзакций
· Precision = 0.89 – из 100 предсказанных мошенничеств 89 реальны
· F1 = 0.83 – оптимальный баланс
2. KNN
· Recall = 0.74 – пропускает больше мошенничеств
· Precision = 0.93 – самый низкий уровень ложных срабатываний
· Хорош, если важна минимизация ложных тревог
3. CatBoost
· Recall = 0.80 – лучший recall среди всех моделей
· Precision = 0.80 – умеренный уровень ложных срабатываний
· Идеален, если важнее найти все мошенничества
4. Decision Tree
· Recall = 0.76 – неплохой показатель
· Precision = 0.53 – много ложных срабатываний
· Простая интерпретируемая модель, но уступает ансамблям
5. Gradient Boosting
· Recall = 0.86 – отличный recall
· Precision = 0.22 – катастрофически низкий precision
· Модель "паникует" и помечает слишком много транзакций как мошеннические
Выводы
1. Для несбалансированных данных критически важно:
   · Использовать стратификацию при разделении выборок
   · Применять балансировку классов (class_weight, scale_pos_weight)
   · Оценивать модели по F1-score, recall и precision, а не по accuracy
2. Ансамблевые методы (XGBoost, CatBoost) показывают лучшие результаты по сравнению с простыми моделями (KNN, дерево решений)
3. XGBoost – оптимальный выбор для данной задачи благодаря:
   · Хорошему балансу precision/recall
   · Высокой скорости обучения
   · Устойчивости к переобучению
4. KNN может быть полезен, если важна минимизация ложных срабатываний (высокий precision)
5. CatBoost стоит выбрать, если приоритет – найти все мошенничества (высокий recall)
